El proyecto es una aplicación de escritorio para la gestión de requisiciones de compra desarrollada como práctica de arquitectura de software, con el propósito de administrar el ciclo de vida completo de requisiciones (creación, edición, envío a revisión, aprobación, rechazo, cancelación), registro de empleados, gestión de permisos por rol y generación de reportes, estructurado en una arquitectura por capas de cinco niveles: CapaPresentación (interfaz de usuario WinForms con formularios como Login, Menú, NuevaRequisicion, EditarRequisicion, MisRequisiciones, AdmRequisiciones, RegistroEmpleados, BuscarProducto, Reportes, que usa FontAwesome.Sharp para iconos y ReportViewerCore/QuestPDF para reportes), CapaNegocio (lógica de negocio con servicios como RequisicionServicio, UsuarioServicio, PermisoServicio, ProveedorServicio, PoliticaImpuestoServicio que implementan patrones de diseño como Builder para construcción de requisiciones, Strategy para cálculo de impuestos (ITBIS estándar, reducido o exento) y State para gestión de estados de requisiciones (Creada, EnRevisión, Aprobada, Rechazada, Cancelada)), CapaDatos (acceso a datos con Dapper y Microsoft.Data.SqlClient para SQL Server, usa procedimientos almacenados y DataSets para reportes, incluye clases como RequisicionDatos, EmpleadoDatos, UsuarioDatos, ArticuloDatos), CapaEntidades (modelos de dominio como Requisicion, Empleado, Usuario, Articulo, Proveedor, Departamento, Evento, PoliticaImpuesto que integran QuestPDF para generación de reportes) y CapaInfraestructura (componentes compartidos de acceso a datos y DataSets), utiliza tecnologías como .NET 9.0 Windows 7.0, C#, WinForms, Dapper, Microsoft.Data.SqlClient, FontAwesome.Sharp 6.6.0, QuestPDF 2025.12.4, ReportViewerCore.WinForms 15.1.33, System.Data.SqlClient, y patrones de diseño para mantenibilidad, sus puntos fuertes incluyen separación clara de responsabilidades por capas, uso de patrones de diseño que facilitan la extensibilidad (Builder para construcción flexible de requisiciones, Strategy para soporte de distintos esquemas de impuestos, State para transiciones controladas de estados de requisiciones), sistema de permisos basado en roles que oculta menús no autorizados, uso de TransactionScope para garantizar integridad de datos en operaciones transaccionales, soporte para múltiples formatos de reportes (ReportViewer, QuestPDF, FastReport .frx) y actualización a .NET 9 para aprovechar mejoras de rendimiento y seguridad.
